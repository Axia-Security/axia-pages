The Insight

Your gate already has the right instinct

The ensemble's conservative bias means it's safe as a filter — but it's sending too much to the LLM. Every over-flagged non-sensitive document is an unnecessary LLM call.

The opportunity: tighten the gate so more noise is filtered before the LLM, without introducing any false negatives.

Same architecture. Better training data. Significantly lower cost.

Enhancement 2 — The Evidence

The three failure patterns in detail

Pattern A: Topic Confusion
"Meeting to discuss Q3 energy trading strategy and regulatory compliance"
Ensemble: SENSITIVE (trading + regulatory)
Reality: NOT SENSITIVE (meeting, no data)

Fix: Add policy docs, compliance discussions, financial planning emails.

Pattern B: Complexity Bias
"Deployed Flyte workflow for k8s pod scaling. Updated IAM roles for staging."
Ensemble: SENSITIVE (technical infra)
Reality: NOT SENSITIVE (routine DevOps)

Fix: Add code reviews, architecture docs, technical RFCs.

Pattern C: Business Assumption
"Updated project timeline for the client deliverable. Review before Friday standup."
Ensemble: SENSITIVE (business content)
Reality: NOT SENSITIVE (project email)

Fix: Add meeting agendas, status updates, project plans.

Next Step

Generate 200+ counter-examples per pattern. Add to training CSV as non-sensitive. Retrain ensemble. Measure gate pass-through rate improvement.

Enhancement 3 — The Evidence

Content-level contrastive pairs

Pair Type A: Policy vs. Data

"Our salary review process follows the annual cycle..." (discussing payroll) vs. "John Smith, SSN 123-45-6789, Annual: $145,000..." (containing payroll data). Different token patterns. Learnable.

Pair Type B: Process vs. Credentials

"Our password policy requires 12+ characters with rotation..." (discussing security) vs. "admin / P@ssw0rd123! / prod-db-east.internal" (containing credentials). Very different structure.

What the Ensemble CAN Learn

Text that discusses a topic uses narrative language, general terms, abstract references. Text that contains actual data has structured fields, specific identifiers, numeric patterns. Transformers see this.

What the Ensemble CANNOT Learn

Who sent it. When they accessed it. Whether they're leaving the company. Where it's going. These signals aren't in the text — they live in metadata the ensemble never sees.

Next Step

Generate contrastive pairs for each of the 21 topic categories in the synthetic data. Each pair: same topic, one discussing it (non-sensitive), one containing real data (sensitive). Add to training CSV.

The Blind Spot

What the ensemble gate can never catch

The Problem

"The garden is blooming, time to harvest."

Zero PII. Zero keywords. Zero structural markers. The ensemble correctly classifies this as Not Sensitive. It skips the LLM entirely. But it's coded insider trading — and nobody catches it.

The same blind spot applies to: a departing employee downloading their normal-looking files at 2 AM, someone using encryption to bypass DLP, bulk exports that look like routine transfers one at a time.

These threats are invisible to any content classifier — no matter how well trained — because the risk signal isn't in the text. It's in the behavior, the timing, the context.

Enhancements 1-5 improve the gate. Enhancement 6 addresses what the gate structurally cannot see.

Enhancement 6 — New

Add a behavioral bypass path

For certain contexts, route communications to the LLM regardless of what the ensemble says.

HR-Flagged Users

Employee on PIP, resignation submitted, or in termination process. All their communications hit the LLM, even routine-looking ones.

UEBA Anomalies

Access at unusual hours, volume spikes, new destinations. When behavioral analytics flag something, bypass the content gate.

High-Value Assets

Any communication touching source code repos, M&A war rooms, or board-level documents. Content classification alone isn't sufficient.

This preserves the cost benefit for the ~90% of normal traffic while ensuring behavioral threats can't slip through the content gate.

The Bigger Picture

Smarter gate + wider training + behavioral path = complete coverage

The ensemble gate has the right instinct but is trained on a narrow world. Expanding to all 32 priority scenarios while tightening its boundaries transforms it from a financial-email classifier into a true enterprise content gate.

And adding a behavioral bypass for high-risk contexts closes the one gap that no amount of training data can fix.

Together, these turn AXIA's classification pipeline from a good content filter into a system that catches threats whether they're visible in the text or hidden in the behavior — across every content domain AXIA is designed to protect.
