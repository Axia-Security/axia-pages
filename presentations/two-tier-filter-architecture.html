How one number in a failed analysis revealed AXIA's most powerful cost advantage

The Problem

Every CISO has the same complaint

DLP systems generate thousands of events per day. Security teams drown in false positives. The real threats get buried under mountains of routine file transfers flagged because they contain a phone number.

The industry's answer has been smarter AI. Better models. More compute.

But what if the answer isn't a bigger brain?

The Analysis

We tested 28,495 training records against AXIA's severity guidelines

The training data classifies by content sensitivity — what PII is present.
AXIA classifies by behavioral risk — why data is moving and where it's going.

The Insight

Conservative error = perfect pre-filter

A classifier that over-flags is annoying. But a classifier that never misses real risk while correctly identifying all the noise? That's a production-grade filter.

It doesn't need to understand context. It doesn't need to assess intent. It just needs to answer one question:

"Is there anything here worth thinking about?"

Two-Tier Architecture

Let the cheap filter handle the noise

Tier 1 — Fast Filter

Lightweight content classifier

No LLM needed. Pattern matching on content signals. If it says "non-sensitive," the interaction is done. No further processing.

~60-70% of all events filtered here
↓
only flagged interactions pass through
↓
Tier 2 — AXIA Judgment Layer

Full behavioral AI analysis

Context, intent, destination, timing, role baselines. This is where AXIA does what nobody else can — understanding why data is moving.

~30-40% of events need real analysis
Making It Sharper

Five enhancements to tighten the filter

1

Relabel known misclassifications

38 samples already identified as non-sensitive. Shift the training boundary immediately.

2

Train against the three failure patterns

Topic confusion, complexity bias, and business-content assumptions. Add counter-examples for each.

3

Build the "about vs. contains" dataset

Content that discusses sensitive topics without containing sensitive data. Highest-impact single enhancement.

4

Scale the non-sensitive baseline

From 24 samples to thousands. Slack chatter, calendar invites, code commits, wiki edits, meeting notes.

5

Preserve the conservative bias

Never optimize precision at the expense of recall. Extra items to Tier 2 = cost issue. Missed threat = security failure.
